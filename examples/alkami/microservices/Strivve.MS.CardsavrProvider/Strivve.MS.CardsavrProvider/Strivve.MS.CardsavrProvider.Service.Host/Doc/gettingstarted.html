Getting Started

Alkami Provider Service


Overview

The provider service template creates a runnable service that includes example settings to illustrate the configurable nature of the service implementation.

These settings should be changed to meet the intended purpose of this service. These setting values are located in the `insert_provider_settings.sql` script and the `ServiceImp.SettingManagement.cs` class.

Read about services

Make sure to read the My First Provider Service tutorial.

Make sure to read the other resources located on the right side panel of this page.

Build the service

Build the solution and make sure the NuGet packages are properly restored and all the projects build correctly.

Configure the service

Run the included `insert_provider_setting.sql` script to insert the service information and the accompanying settings.

Updating Packages

It's always good to stay up to date with Alkami's latest packages. Update any stale Alkami packages using the NuGet Package Manager.

  2. My First Provider Service
    Using the template, create an example service to learn about the development patters. It demonstrates how to update existing settings, add new settings, create validations, add new public contract methods and testing the service.

    Eventually, you'll want to connect a widget to a microservice. This will teach the developer how to add code to the widget in order to connect with the service.

    The service will need to be packaged using Chocolatey so that it can be installed to a given environment, this guide will teach the developer how to do that. The installer has already been included in the project so skip to "Configuring a custom .nuspec file" to complete the packaging process.
